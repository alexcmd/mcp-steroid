# 0.95.0 Release Social Posts

## LinkedIn Post

IntelliJ for AI Agents, not humans — MCP Steroid v0.95.0 is out.

What's new:

1. McpEditingGuard ends the memory-vs-disk modal blocker. Every `steroid_execute_code` and `steroid_apply_patch` call now flows through one guard: modality fail-fast, pre-flight commit + saveAllDocuments + awaitRefresh, prefer-disk reload while the body runs, post-flight refresh. The "memory differs from disk" dialog can no longer freeze the IDE while an agent is editing — issue #27.

2. New typed PSI recipe `mcp-steroid://ide/find-duplicates`. Agents were resorting to private-field reflection to extract clone pairs from IntelliJ's `DuplicatedCode` inspection — because they probed the wrong index for the class, concluded it was unavailable, and went around it. The class is on the script classpath already; a direct import works. Reflection policy now stated in the root prompts every agent reads. Issue #33.

3. Three agent-first design tenets codified. New canonical statement at docs/PHILOSOPHY.md, mirrored at runtime as `mcp-steroid://skill/design-philosophy`. The MCP tool surface stays narrow on purpose (10 tools, listed by name); power lives in prompts and direct IntelliJ APIs called from `steroid_execute_code`; `McpScriptContext` methods are last-resort. Per-folder agent guides and the README now cite the tenets in their own scope, so the strategy isn't only in one file anymore.

4. Plugin Verifier still reports zero internal-API usages on both IntelliJ 2025.3 and 2026.1. Forward compatibility holds.

The evals repo is open: github.com/jonnyzzz/mcp-steroid. 10 MCP tools, a `mcp-steroid://` prompt resource library, Docker integration tests for Claude, Codex, and Gemini. Contribute a scenario, run the evals, improve a skill.
